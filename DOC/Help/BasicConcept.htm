к содержанию

Весь функционал основан на том, что рабочий процесс для работников можно разбить на простые операции перемещения и действия с объектами. Очерёдность и обязательность этих действий и перемещений может настраиваться и меняться в процессе работы, зависит от конкретного рабочего процесса.

Основные понятия

·         Пользователь – учётная запись пользователя – имя, пароль, дополнительные сведения подключения для человека работающего в данной среде.

·         Группа пользователей служит для установки единых прав для выполнения действий и перемещений, пользователи должны быть добавлены в группы для получения этих прав. Есть четыре предопределённые группы:

o    Guest – гость, только просмотр сведений в БД;

o    User – пользователь, позволяет выполнять перемещения и действия;

o    ObjDesigner – позволяет создавать объекты;

o    TypeDesigner – позволяет создавать классы, действия, определять, разрешения для перемещений и действий;

o    Admin – позволяет создавать группы и пользователей;

 

·         Свойство – любое значение, используемое и в процессе работы, имеет разные типы (текст, число, дата и т.п.) и смысловое название. (например: свойство с названием «длина (м.)» может иметь дробное значение «5.67»)

 

·         Действие – любая операция, производимая с объектом, например: ремонт, калибровка, обслуживание, введение в эксплуатацию. Включает в себя свойства, которые меняются/описываются во время выполнения (например: при выполнении действия «покраска» изменится значение свойства «цвет»)

 

·         Класс (тип) –служит для обобщения объектов по общим свойствами, обозначает группы реальных или абстрактных объектов (например: класс «монитор» является абстрактным, а «Dell 2209wa» - обозначает конкретный класс оборудования. )

o    Абстрактные – используется в иерархии классов для обобщения классов (например: абстрактный класс «Монитор»). Не имеют экземпляров.

o    Номерные – для оборудования имеющего серийные/инвентарные номера (например: монитор «Dell 2209wa» с серийным номером 1234). Можно добавлять экземпляры с заводским номером в качестве имени.

o    Целочисленные – для расходных материалов и запасных частей (например: «карандаш» - исчисляется в штуках.). Можно добавлять экземпляры с указанием количества, в качестве названия можно указать дату поступления или любой другой идентификатор партии.

o    Дробные – для расходных материалов и запасных частей которые измеряются в килограммах, литрах, метрах и т.п. (например: смазка Литол-24 исчисляется в килограммах ). Можно добавлять экземпляры с указанием количества, в качестве названия можно указать дату поступления или любой другой идентификатор партии.

 

·         Объект – отражение реального объекта, с которым можно производить действия и перемещения

o     номерной объект (например: монитор «Dell 2209wa» с серийным номером 1234 → [Dell 2209wa]1234)

o     целочисленный объект – например «карандаш -23шт.»

o    дробный объект – например «Литол24 -2.56кг.»

 

 

·         Иерархия (дерево) классов — аналог файловой системы, в которой абстрактные типы являются «папками», а номерные, целочисленные и дробные классы - «файлами». По аналогии с файловой системой для любого класса в иерархии можно задать разрешения и действия, которые будут наследоваться вложенными классами.

·         Иерархия местоположения — такое же дерево как и дерево классов, но объекты в нем сгруппированы не по типам а по местоположению.
